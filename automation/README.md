# 휴가증 자동 반영 — 서무 담당자용 자동화

COSMAX COIN 그룹웨어(coin.cosmax.com)에 휴가증을 자동으로 **임시저장**하는 도구입니다.

> 자동화는 임시저장까지만 진행하며, **실제 신청은 서무 담당자가 검토 후 직접 진행**합니다.

## 📦 구성 파일

| 파일 | 역할 |
|---|---|
| `setup.bat` | **최초 1회 실행** — Python 패키지 + Edge 드라이버 설치 |
| `run.bat` | **평소 사용** — 더블클릭으로 자동화 실행 |
| `automate.py` | 메인 자동화 스크립트 (Python + Playwright) |
| `requirements.txt` | 필요한 Python 패키지 목록 |
| `profile/` | (자동 생성) Edge 영구 프로필 — 쿠키 보관 |
| `last_error.log` | (자동 생성) 마지막 실행 에러 로그 |
| `error_*.png/html`, `after_save_*.png` | (자동 생성) 디버그용 — 문제 발생 시 분석 자료 |

## 🚀 최초 설치 (한 번만)

### 1. Python 설치
- https://www.python.org/downloads/ 에서 **Python 3.10~3.12** (3.13/3.14는 일부 호환성 이슈 가능)
- 설치 화면에서 **"Add Python to PATH"** 옵션을 반드시 체크
- 설치 완료 후 PC 재시작 권장

### 2. 본 폴더에서 `setup.bat` 더블클릭
- Playwright + Edge 드라이버가 자동 설치됩니다 (1~3분)
- 설치 화면에 [완료] 메시지가 뜨면 성공

### 3. 첫 실행 (로그인 등록)
- `run.bat` 더블클릭
- 파일 선택 다이얼로그가 뜨면 **취소** (또는 JSON 파일 있으면 선택)
- Edge가 자동으로 열리면서 그룹웨어 로그인 화면 표시
- **사번 / 비밀번호로 직접 로그인** → 로그인 완료 후 자동 진행
- **이후 실행부터는 로그인이 자동 유지됩니다** (Edge 영구 프로필 활용)

## 📋 평소 사용 절차

1. 현장에서 작업자들이 휴가증 사이트에 입력
   - 사이트: https://djlee-cosmax.github.io/Vacation-Permit-Auto-System/
2. 작성된 휴가증이 충분히 모이면 **`파일로 내보내기`** 클릭
   - JSON 파일(자동화 입력용) + XLSX 파일(검토용) 동시 다운로드
3. 본 폴더의 `run.bat` 더블클릭
4. 파일 선택 다이얼로그에서 방금 다운로드한 **JSON 선택**
5. 확인 메시지 → 예
6. Edge가 자동으로 신청서를 차례로 임시저장
7. 완료 알림이 뜨면 그룹웨어 좌측의 신청 내역에서 임시저장된 신청서들 확인
8. **XLSX 파일을 참고하여 검토** 후 그룹웨어에서 직접 신청

## ⚠️ 결과 해석

자동화 종료 시 결과 알림:

| 알림 | 의미 |
|---|---|
| `완료 — N건 모두 임시저장 완료` | 정상 |
| `완료 (누락자 있음)` | 임시저장은 됐지만 일부 작업자가 그룹웨어 명단에 없어 빠짐 — XLSX와 비교 |
| `일부 신청서 실패` | 잔여일수 부족 등 그룹웨어 검증 거부. 해당 작업자는 별도 처리 필요 |

특히 누락 작업자가 있는 경우:
- 그룹웨어 인사 명단에 해당 사번이 없거나
- 신청서 작성 권한이 없거나
- 휴가증 사이트의 작업자 명단이 오래된 경우

→ 휴가증 사이트의 `작업자 명단` 모달에서 정보를 확인하고 조정해 주세요.

## 🔧 주요 시나리오별 대응

### 작업자 명단 변경 (입사·퇴사 등)
- 휴가증 사이트 → 우측 상단 `작업자 명단` → `직접 추가` 또는 엑셀 업로드
- 본인 PC에만 반영됨. 사이트 전체 명단(`workers.json`)을 갱신하려면 담당자에게 문의

### 그룹웨어 비밀번호 변경
- `run.bat` 실행 시 자동 로그인 실패 → 로그인 화면 표시됨
- 새 비밀번호로 직접 로그인 → 이후 자동 유지
- 또는 `profile/` 폴더를 통째로 삭제하고 다시 첫 실행 절차 진행

### 그룹웨어 화면이 변경됐을 때
- 자동화가 특정 단계에서 멈춤
- 폴더의 `error_*.png`, `last_error.log` 파일을 담당자(이동준)에게 전달

## ❓ 트러블슈팅

| 증상 | 해결 |
|---|---|
| `setup.bat`에서 Python 없음 에러 | Python 설치 후 PC 재시작. PATH 옵션 체크 확인 |
| `setup.bat`에서 패키지 설치 실패 | 사내 네트워크 프록시 문제. IT팀 또는 담당자에게 문의 |
| `run.bat`에서 Edge가 안 열림 | `setup.bat` 다시 실행해서 Edge 드라이버 재설치 |
| 로그인 후에도 자동화가 진행 안 됨 | Edge 창을 그대로 두고 `run.bat`을 다시 실행 |
| 일부 신청서만 등록 실패 | 결과 알림의 실패 사유 확인 (잔여일수 부족 등) |
| 누락 작업자 알림이 떴음 | 휴가증 사이트의 명단과 그룹웨어 명단 비교 |
| 자동화 실행 중 그룹웨어가 멈추거나 느림 | 네트워크 상황 확인 후 재시도 |
| 임시저장 결과를 그룹웨어에서 못 찾겠음 | 그룹웨어 좌측 메뉴 `근태/휴가신청` 클릭, 상단의 신청 내역 표 확인 |

## 🔄 디버그 파일 정리

폴더에 `error_*`, `after_save_*` 파일이 많이 쌓이면 폴더가 무거워집니다.
가끔 다음 명령으로 정리하셔도 됩니다 (PowerShell):

```powershell
cd "C:\Users\djlee\cosmax-vacation-automation"  # 본인 폴더 경로
Remove-Item error_*.png, error_*.html, after_save_*.png -ErrorAction SilentlyContinue
```
