1. 소개서

현장에서 작성한 휴가증을 그룹웨어 COSMAX COIN에 자동으로 임시저장하는 도구입니다. 현장 인원이 디지털로 작성 → 서무 담당자가 일괄 자동 등록 → 검토 후 신청하는 흐름입니다.

1.1 프로그램 목적

  • 기존 종이 휴가증 → 디지털 양식으로 전환
  • 서무 담당자가 그룹웨어에 매 건 수동 입력하던 작업을 자동화
  • 휴가 유형·기간·작업장 별로 신청서를 정확히 분류하여 등록
  • 한국 공휴일·잔여일수 부족 등 검증 오류를 사전·사후에 안내

1.2 전체 흐름

  1. 현장 — 작업자가 휴가증 사이트에 자신의 휴가 내역 작성
  2. 현장 — 작성 완료 후 "파일로 내보내기" 클릭 (JSON + XLSX 다운로드)
  3. 전달 — JSON / XLSX 파일을 서무 담당자에게 전달 (USB, 메일, 공유폴더 등)
  4. 서무 — 자동화 프로그램(`run.bat`)에 JSON 파일을 넣고 실행
  5. 그룹웨어 — 프로그램이 자동으로 신청서를 임시저장
  6. 서무 — XLSX 파일과 그룹웨어 내역을 비교 검토 → 직접 신청 처리

1.3 주요 특징

기능설명
유형별 다중 선택작업자별로 휴가증 1건씩 입력 (한 사람 여러 유형은 여러 휴가증)
9가지 휴가 유형연차·반차(오전/오후)·반반차(오전/오후)·생휴·결근·결근(오전/오후)
자동 그룹화생휴 / 비생휴 두 분류, 같은 작업자 중복 시 자동 분리
한국 공휴일 자동 제외실수로 공휴일에 휴가증 작성해도 자동 빠짐
사번 기반 검색동명이인이 있어도 사번으로 정확히 매칭
실패 사유 표시잔여일수 부족 등 그룹웨어 응답을 그대로 알림
작업장별 파일명화성/평택 다수 인원 기준으로 파일명에 자동 표기
데이터 백업/복원휴가증과 작업자 명단을 JSON으로 백업

1.4 구성 요소

웹 사이트 (휴가증 작성용)

  • URL: https://djlee-cosmax.github.io/Vacation-Permit-Auto-System/
  • 현장 작업자가 PC/모바일 브라우저로 접속
  • GitHub Pages 호스팅 — 24시간 가용

자동화 프로그램 (서무 PC 전용)

  • 서무 담당자 PC에 설치
  • Python + Playwright + Microsoft Edge
  • 그룹웨어(coin.cosmax.com)에 자동 등록

2. 매뉴얼 (현장용)

현장 작업자가 휴가증을 작성하고 서무에게 전달하기까지의 절차입니다.

2.1 사이트 접속

브라우저 (Edge / Chrome 등) 에서 다음 주소로 접속합니다.

https://djlee-cosmax.github.io/Vacation-Permit-Auto-System/

즐겨찾기에 추가해 두면 매번 입력할 필요가 없습니다.

참고: 사이트는 사내 네트워크가 아니어도 접속 가능합니다 (GitHub Pages). 단, 작업자 명단·작성한 휴가증은 본인 PC 브라우저에만 저장됩니다.

2.2 휴가증 작성 절차

  1. 이름 입력 — 글자를 입력하면 작업자 명단에서 자동완성됩니다. 본인 이름 선택 시 사번/근무지/연락처가 자동 채워집니다.
  2. 구분 선택 — 휴가 유형 (연차 / 반차 / 반반차 / 생휴 / 결근 등)
  3. 개수 입력 — 연차 2개, 반차 1개 등. 합계 일수가 자동 계산되어 표시됩니다.
  4. 기간 선택 — 시작일~종료일. 연차 2개라면 보통 2일 연속.
  5. 사유 입력 — 휴가 사유
  6. 연락처 입력 — 010-0000-0000 형식 (자동 하이픈)
  7. [+ 추가] 클릭 — 작성된 휴가증 카드가 우측에 추가됩니다.
주의: 한 작업자가 여러 유형(연차+반차 등)을 사용할 때는 휴가증을 유형별로 따로 작성합니다. 예: 김철수 연차 2개 + 반차 1개 → 휴가증 2건 작성.

2.3 휴가 유형 안내

구분1개당 일수출퇴근 안내
연차1.0일종일 휴가
반차(오전)0.5일오후 12시 50분 출근
반차(오후)0.5일오후 12시 퇴근
반반차(오전)0.25일오전 10시 출근
반반차(오후)0.25일오후 3시 퇴근
생휴1.0일생리 휴가
결근1.0일연차 없이 종일 출근 못할 때
결근(오전)0.5일오후 12시 50분 출근 (반차 잔여 없을 때)
결근(오후)0.5일오후 12시 퇴근 (반차 잔여 없을 때)

목록에 없는 휴가 유형(예: 경조사·교육 등)은 서무 담당자에게 직접 요청해 주세요.

2.4 작업자 명단 관리

우측 상단 [작업자 명단] 버튼으로 명단을 확인·수정합니다.

  • 파일 업로드 — 엑셀 명단을 일괄 가져오기 (헤더: 이름 / 사번 / 근무지 / 연락처)
  • + 직접 추가 — 신규 작업자 한 명씩 추가
  • 기본 명단으로 재설정 — 서버에 등록된 기본 명단(63명)으로 되돌리기

2.5 파일 내보내기

휴가증 작성을 마치면 우측 상단 [파일로 내보내기] 클릭. 다음 2개 파일이 동시에 다운로드됩니다.

파일용도
휴가증_2026-05-20 (화성).json서무 자동화 프로그램 입력용
휴가증_2026-05-20 (화성).xlsx서무 검토용 (신청서별 / 원본 입력 2시트)

파일명 끝의 (화성) / (평택)은 작성된 휴가증의 다수 근무지가 자동으로 표기됩니다. 두 파일 모두 서무 담당자에게 전달해 주세요.

2.6 데이터 백업/복원

작성한 휴가증·작업자 명단은 브라우저(localStorage)에 저장되어 있어서, 캐시 삭제나 시크릿 모드 사용 시 사라질 수 있습니다.

  • [백업] — 작업자 명단 + 작성된 휴가증 전체를 JSON 파일로 저장
  • [복원] — 저장한 백업 파일로 복원 (현재 데이터는 덮어쓰여집니다)

중요한 작업 전·후에 한 번씩 백업해 두는 것을 권장합니다.

2.7 트러블슈팅 (현장)

증상해결
이름 검색이 안 됨작업자 명단에 이름이 등록되어 있는지 확인. 없으면 + 직접 추가
휴가증 작성 후 다른 PC에서 안 보임localStorage는 PC별로 독립. 백업 → 다른 PC에서 복원
파일 다운로드가 안 됨브라우저의 팝업/다운로드 차단 설정 확인
실수로 휴가증 삭제최근 백업 파일 있으면 복원, 없으면 다시 작성

3. 매뉴얼 (서무용)

현장에서 받은 휴가증 JSON 파일을 자동화 프로그램에 넣어 그룹웨어에 일괄 임시저장하고, 검토 후 신청까지 진행하는 절차입니다.

3.1 최초 설치 (한 번만)

(1) Python 설치

  • https://www.python.org/downloads/ 에서 Python 3.10~3.12 다운로드
  • 설치 화면에서 "Add Python to PATH" 옵션 반드시 체크
  • 설치 완료 후 PC 재시작 권장

(2) 프로그램 폴더 배치

  • USB / 공유폴더에서 받은 자동화 폴더를 적당한 위치에 복사
  • 예: D:\휴가증자동화\

(3) setup.bat 실행

  1. 폴더 안의 setup.bat 더블클릭
  2. Playwright + Edge 드라이버가 자동 설치 (1~3분)
  3. "Setup Complete" 메시지가 뜨면 성공

(4) 첫 실행 (로그인 등록)

  1. run.bat 더블클릭
  2. 파일 선택 다이얼로그가 뜨면 취소 (또는 테스트 JSON 선택)
  3. Edge가 자동으로 열리면서 그룹웨어 로그인 화면 표시
  4. 서무 사번/비밀번호로 직접 로그인
  5. 이후 실행부터는 자동 로그인 유지
참고: 영구 프로필 폴더(profile/)에 쿠키가 저장되어 다음 실행부터 자동 로그인됩니다. 비밀번호 변경 시에는 다시 한 번 로그인이 필요합니다.

3.2 평소 사용 절차

  1. 현장에서 받은 JSON 파일을 다운로드 폴더 등에 저장
  2. 자동화 폴더의 run.bat 더블클릭
  3. 파일 선택 다이얼로그에서 방금 받은 JSON 선택
  4. "신청서 N건 / 인원 N명 / 총 N일 ... 진행하시겠습니까?" 확인 → 예
  5. Edge가 자동으로 신청서를 차례로 임시저장
  6. 완료 알림이 뜨면 결과 확인
  7. 그룹웨어 좌측 신청 내역에서 임시저장된 항목들 확인
  8. XLSX 파일과 비교 검토 후 그룹웨어에서 직접 신청

3.3 결과 해석

알림 제목의미대응
완료 N건 모두 임시저장 성공 그룹웨어 검토 후 신청
완료 (누락자 있음) 임시저장은 성공했지만 일부 작업자가 그룹웨어 명단에 없어 빠짐 XLSX와 비교, 빠진 작업자는 별도 처리
일부 신청서 실패 잔여일수 부족, 중복 등 그룹웨어 검증 거부 알림에 표시된 실패 사유 확인 후 별도 처리

3.4 주요 시나리오 대응

잔여일수 부족 (가장 흔한 케이스)

그룹웨어가 "잔여일수를 초과할 수 없습니다" 등의 메시지를 반환하면 해당 신청서가 실패 처리됩니다.

  • 해당 작업자의 실제 잔여일수 확인
  • 잔여일수 내에서만 신청 가능한지 협의
  • 또는 그룹웨어에서 수동으로 별도 처리

그룹웨어 명단에 없는 작업자

알림에 "그리드에서 찾지 못한 작업자" 섹션이 표시되면:

  • 해당 사번이 그룹웨어 인사 명단에 있는지 확인
  • 휴가증 사이트의 작업자 명단이 오래된 경우 → 담당자에게 갱신 요청

그룹웨어 비밀번호 변경

  • run.bat 실행 시 자동 로그인 실패 → 로그인 화면 표시
  • 새 비밀번호로 직접 로그인 → 이후 자동 유지
  • 완전 초기화가 필요하면 profile/ 폴더 삭제 후 첫 실행 절차 재진행

그룹웨어 화면이 변경됐을 때

자동화가 특정 단계에서 멈추면 셀렉터 조정이 필요할 수 있습니다.

  • 폴더의 error_*.png, last_error.log를 담당자에게 전달
  • GitHub 저장소에서 최신 버전 받기
