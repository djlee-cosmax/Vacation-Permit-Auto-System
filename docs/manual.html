1. 소개서

작업자가 본인 휴대폰으로 직접 휴가증을 작성하면 클라우드에 자동 저장되고, 서무 담당자는 매일 한 번의 클릭으로 그날의 모든 휴가증을 일괄 취합하여 그룹웨어 COSMAX COIN에 자동 임시저장합니다.

1.1 프로그램 목적

  • 기존 종이 휴가증 → 디지털 양식으로 전환 (작성 주체는 그대로 개인)
  • 서무 담당자가 그룹웨어에 매 건 수동 입력하던 작업을 자동화
  • 작업자가 작성한 휴가증이 클라우드에 자동 누적 — 조장·서무가 별도 취합 불필요
  • 휴가 유형·기간·작업장 별로 신청서를 정확히 분류하여 등록
  • 한국 공휴일·잔여일수 부족 등 검증 오류를 사전·사후에 안내

1.2 전체 흐름

  1. 작업자 (모바일) — 본인 휴대폰으로 사이트 접속 → 휴가증 작성 → "+ 추가" 클릭
  2. 클라우드 자동 저장 — 작성된 휴가증이 Firebase 클라우드에 미처리 상태로 즉시 업로드
  3. 서무 (PC) — 사이트(?leader=1) 접속 → "서버에서 불러오기" 클릭 (미처리 휴가증 + 최근 7일 자동 취합)
  4. 서무 (PC) — 휴가증 검토 → "파일로 내보내기" → JSON + XLSX 다운로드
  5. 서무 (PC) — 자동화 프로그램(run.bat)에 JSON 입력 → 자동 임시저장
  6. 서무 — 그룹웨어 신청 내역 + XLSX 비교 검토 → 직접 신청 처리
  7. 서무 (PC) — 사이트로 돌아와 "처리 완료" 클릭 → 클라우드에서 처리 완료로 마크 (다음 불러오기 제외)
개선 효과: 조장이 작업자별 휴가 정보를 받아 사이트에 일일이 입력하는 과정이 사라지고, 작업자 본인이 자기 휴대폰으로 직접 작성하게 됩니다. 서무는 사이트에서 자동 취합된 휴가증을 받아 자동화 프로그램에 입력만 하면 됩니다.

2. 매뉴얼 (작업자용)

작업자가 본인 휴대폰(또는 PC)으로 직접 휴가증을 작성하고, 작성된 휴가증은 자동으로 클라우드에 저장됩니다. 별도 파일 전달이나 조장에게 보내는 과정 없이 서무가 그날 휴가증을 일괄 가져갑니다.

2.1 사이트 접속 (모바일·PC)

휴대폰 또는 PC 브라우저(Edge / Chrome / Safari 등)에서 다음 주소로 접속합니다.

https://djlee-cosmax.github.io/Vacation-Permit-Auto-System/

휴대폰 홈 화면에 추가하거나 즐겨찾기에 등록해 두면 편리합니다.

참고: 인터넷만 연결되어 있으면 어디서든 접속 가능합니다 (회사 와이파이 / LTE / 가정 와이파이 모두 OK). 작성된 휴가증은 클라우드에 자동 저장됩니다.

2.2 로그인

사이트 처음 진입 시 로그인 화면이 표시됩니다.

  1. 사번 입력 (본인의 9자리 사번)
  2. 비밀번호 입력 — 최초 비밀번호: 1234
  3. [로그인] 클릭
  4. "OOO님 환영합니다" 토스트 표시 → 메인 화면 진입
참고: 한 번 로그인하면 30일간 자동 유지됩니다. 매번 다시 입력할 필요 없어요. 다른 사람이 본인 휴대폰을 사용해야 한다면 우상단 [로그아웃] 클릭 후 새로 로그인.
로그인 실패 시:
  • "등록되지 않은 사번입니다" — 작업자 명단에 본인 사번이 없음. 관리자(이동준)에게 명단 추가 요청
  • "비밀번호가 일치하지 않습니다" — 기본 비밀번호 1234 다시 확인

2.3 앱처럼 홈 화면에 추가하기 (권장)

매번 브라우저에서 주소를 입력하는 대신, 사이트를 휴대폰 홈 화면에 앱처럼 추가하면 아이콘 한 번 탭으로 바로 열 수 있습니다. 일반 앱과 동일한 풀스크린 화면으로 동작합니다.

iOS (아이폰 · Safari)

  1. Safari로 사이트 접속
  2. 화면 하단 가운데 공유 버튼 (⬆️ 위쪽 화살표가 있는 네모) 탭
  3. 메뉴 아래쪽으로 스크롤 → "홈 화면에 추가" 선택
  4. 이름 확인 (기본: "휴가증") → 우측 상단 "추가" 탭
  5. 홈 화면에 COSMAX 로고 아이콘 생성 완료

Android (안드로이드 · Chrome)

  1. Chrome으로 사이트 접속
  2. 화면 우상단 ⋮ (점 세 개) 메뉴 탭
  3. "홈 화면에 추가" 또는 "앱 설치" 선택
  4. "설치" 또는 "추가" 확인
  5. 홈 화면에 아이콘 생성 완료
참고: 일부 안드로이드 기기는 사이트 접속 시 화면 하단에 "앱 설치하시겠습니까?" 배너가 자동으로 표시되기도 합니다. 그때 바로 설치하셔도 됩니다.

추가된 아이콘을 탭하면 일반 앱처럼 풀스크린으로 실행됩니다 (주소창 없음). 자동 업데이트되므로 별도 관리도 필요 없습니다.

2.4 휴가증 작성 절차

  1. 이름 입력 — 글자를 입력하면 작업자 명단에서 자동완성됩니다. 본인 이름 선택 시 근무지가 자동 채워집니다.
  2. 구분 선택 — 휴가 유형 (연차 / 반차 / 반반차 / 생휴 / 하기휴가 / 결근 등)
  3. 개수 입력 — 연차 2개, 반차 1개 등. 합계 일수가 자동 계산되어 표시됩니다.
  4. 기간 선택 — 시작일~종료일. 연차 2개라면 보통 2일 연속.
  5. 사유 입력 — 휴가 사유
  6. 연락처 입력 — 010-0000-0000 형식. 본인 휴대폰의 마지막 4자리가 자동 식별자로 사용됩니다.
  7. [+ 추가] 클릭 — 작성된 휴가증 카드가 우측에 추가되고 클라우드에 자동 저장됩니다.
완료! 작성된 휴가증은 클라우드에 즉시 저장되며, 서무 담당자가 매일 자동으로 가져갑니다. 본인이 따로 파일을 보내거나 조장에게 전달할 필요가 없습니다.
주의: 한 작업자가 여러 유형(연차+반차 등)을 사용할 때는 휴가증을 유형별로 따로 작성합니다. 예: 김철수 연차 2개 + 반차 1개 → 휴가증 2건 작성.

2.6 내 휴가증 조회·취소

본인이 작성한 휴가증은 상단 [내 휴가증] 버튼으로 언제든 조회·취소할 수 있습니다.

  1. 상단 [내 휴가증] 버튼 클릭
  2. 이름 + 휴대폰 마지막 4자리 입력 (이미 휴가증을 작성한 휴대폰이면 자동 채움)
  3. [조회] 클릭 → 최근 14일 내 본인이 작성한 휴가증 목록 표시
  4. 잘못 작성한 휴가증은 [취소] 버튼 → 확인 → 삭제
핵심 동작 — 삭제 시 클라우드 동기화:
  • 화면의 휴가증 카드 우측 [삭제] 또는 [내 휴가증] 모달의 [취소]를 누르면, 본인 화면뿐 아니라 클라우드(서버)에서도 즉시 삭제됩니다.
  • 따라서 서무가 [서버에서 불러오기] 했을 때 사라진 휴가증은 가져오지 않습니다.

서무 처리 시점별 영향

삭제 시점영향
서무 [서버에서 불러오기] 전 가장 안전. 서무는 사라진 휴가증을 알 수 없음 → 자연스럽게 처리에서 빠짐
서무 [서버에서 불러오기] 후 / 자동화 실행 전 클라우드는 사라지지만 서무 PC 화면에는 남아 있음 → 서무가 작업자에게 확인 후 화면에서 [삭제]로 정리하거나 그대로 진행
서무 자동화 실행 후 (그룹웨어 등록 완료) 그룹웨어 신청서는 이미 등록된 상태 → 서무가 그룹웨어에서 별도 취소 처리 필요. 가능하면 자동화 전에 알려주세요
주의 1 (휴대폰 인증): 휴가증을 작성한 휴대폰에서만 본인이 직접 [취소] 할 수 있습니다. 다른 휴대폰에서 본인 휴가증 조회는 가능하지만 🔒 표시가 되어 취소는 안 됩니다. 휴대폰을 바꾼 경우 새 휴가증은 새 휴대폰에서 작성하시고, 옛 휴가증은 서무에게 요청해 주세요.
주의 2 (가능한 한 빨리): 잘못 작성한 휴가증은 서무가 자동화를 돌리기 전에 빨리 삭제해 주세요. 자동화 후에 삭제하면 클라우드는 정리되지만 그룹웨어에는 이미 등록된 상태가 됩니다.

2.7 작업자 명단 조회

상단 [작업자 명단] 버튼으로 등록된 작업자 명단을 확인할 수 있습니다 (조회 전용).

  • 이름 / 근무지 / 연락처 표시 (사번은 비공개)
  • 검색창에서 이름·근무지·연락처로 검색
  • 이름 가나다순 자동 정렬

3.1 서무 사번 로그인

서무 담당자는 본인 사번으로 로그인하면 자동으로 서무 권한이 부여됩니다.

  • 사이트 URL: https://djlee-cosmax.github.io/Vacation-Permit-Auto-System/
  • 로그인 정보 — 사번 122240096 (김가영) / 비밀번호 1234
  • 한 번 로그인하면 30일간 자동 유지
서무 사번으로 로그인 시 추가로 보이는 기능:
  • 우측 카드에 [서버에서 불러오기] 버튼 — 미처리 휴가증 일괄 취합 (최근 7일)
  • 우측 카드에 [처리 완료] 버튼 — 처리 후 클라우드에서 마크
  • 상단에 [파일로 내보내기] 버튼 — JSON + XLSX 다운로드
주의: 일반 작업자 사번으로 로그인하면 서무 기능이 표시되지 않습니다. 반드시 서무 사번(122240096)으로 로그인해 주세요. 모바일에서는 어떤 사번이든 작업자 화면만 표시됩니다 — 서무 작업은 PC에서 진행하세요.

3.2 매일 사용 절차 (사이트 → 자동화 → 처리 완료)

  1. 본인 PC에서 사이트 접속 (서무 모드 URL)
  2. 우측 카드의 [서버에서 불러오기] 클릭 → "서버에서 미처리 휴가증 N건을 발견했습니다" 확인 → 예
  3. 화면에 표시된 휴가증 목록 검토 (잘못된 입력 / 누락 등 확인)
  4. 잘못된 휴가증이 있으면 작성자에게 취소 요청 (또는 본인이 [내 휴가증]에서 취소)
  5. 상단 [파일로 내보내기] 클릭 → JSON + XLSX 두 파일 자동 다운로드
  6. 자동화 폴더의 run.bat 더블클릭
  7. 파일 선택 다이얼로그에서 방금 다운로드한 JSON 선택
  8. "신청서 N건 / 인원 N명 / 총 N일 ... 진행하시겠습니까?" 확인 → 예
  9. Edge가 자동으로 신청서를 차례로 임시저장
  10. 완료 알림 확인 → 그룹웨어 신청 내역에서 임시저장 결과 확인
  11. XLSX 파일과 비교 검토 후 그룹웨어에서 직접 신청
  12. 모든 처리가 끝나면 사이트로 돌아와 우측 카드의 [처리 완료] 클릭 → 클라우드에 마크
핵심 동작 — 처리 완료 마크:
  • [처리 완료]를 누른 휴가증은 클라우드에서 processed=true로 마크됩니다
  • 다음 [서버에서 불러오기] 시 처리 완료된 휴가증은 더 이상 가져오지 않습니다 (중복 방지)
  • [처리 완료]를 누르지 않으면 다음 날에도 불러올 수 있어 안전합니다 (자동화 실패 / 늦은 저녁 작성 / 주말 누적 등)
주의: 자동화 실행 + 그룹웨어 최종 신청까지 모두 완료된 후에 [처리 완료]를 누르세요. 그 전에 누르면 다시 불러오기 어려워질 수 있습니다.

3.3 자동화 최초 설치 (한 번만)

(1) Python 설치

  • https://www.python.org/downloads/ 에서 Python 3.10~3.12 다운로드
  • 설치 화면에서 "Add Python to PATH" 옵션 반드시 체크
  • 설치 완료 후 PC 재시작 권장

(2) 프로그램 폴더 배치

  • USB / 공유폴더에서 받은 자동화 폴더(또는 ZIP)를 적당한 위치에 복사
  • 예: D:\휴가증자동화\
  • ZIP 파일은 압축 풀기 전 우클릭 → 속성 → "차단 해제" 체크 → 확인

(3) setup.bat 실행

  1. 폴더 안의 setup.bat 더블클릭
  2. Playwright + Edge 드라이버가 자동 설치 (1~3분)
  3. "Setup Complete" 메시지가 뜨면 성공

(4) 첫 실행 (로그인 등록)

  1. run.bat 더블클릭
  2. 파일 선택 다이얼로그가 뜨면 취소 (또는 테스트 JSON 선택)
  3. Edge가 자동으로 열리면서 그룹웨어 로그인 화면 표시
  4. 서무 사번/비밀번호로 직접 로그인
  5. 이후 실행부터는 자동 로그인 유지
참고: 영구 프로필 폴더(profile/)에 쿠키가 저장되어 다음 실행부터 자동 로그인됩니다. 비밀번호 변경 시에는 다시 한 번 로그인이 필요합니다.

3.4 주요 시나리오 대응

잔여일수 부족 (가장 흔한 케이스)

그룹웨어가 "잔여일수를 초과할 수 없습니다" 등의 메시지를 반환하면 해당 신청서가 실패 처리됩니다.

  • 해당 작업자의 실제 잔여일수 확인
  • 잔여일수 내에서만 신청 가능한지 협의
  • 또는 그룹웨어에서 수동으로 별도 처리

그룹웨어 명단에 없는 작업자

알림에 "그리드에서 찾지 못한 작업자" 섹션이 표시되면:

  • 해당 사번이 그룹웨어 인사 명단에 있는지 확인
  • 휴가증 사이트의 작업자 명단이 오래된 경우 → 담당자에게 갱신 요청

그룹웨어 비밀번호 변경

  • run.bat 실행 시 자동 로그인 실패 → 로그인 화면 표시
  • 새 비밀번호로 직접 로그인 → 이후 자동 유지
  • 완전 초기화가 필요하면 profile/ 폴더 삭제 후 첫 실행 절차 재진행

그룹웨어 화면이 변경됐을 때

자동화가 특정 단계에서 멈추면 셀렉터 조정이 필요할 수 있습니다.

  • 폴더의 error_*.png, last_error.log를 담당자에게 전달
  • GitHub 저장소에서 최신 버전 받기
